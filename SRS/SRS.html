Software Requirements Specification



  1. Introduction

    1. Purpose
      The purpose of Fortune is to provide venture capitals (VCs) with the required data to determine whether a startup will succeed or not, while utilizing machine learning techniques. The objective of this project is to decrease the time taken for VCs to reach a final decision on whether to invest in a startup or not. By using existing machine learning algorithms, our main aim is to provide an application that can be used to accelerate the decision making process of VCs.
    2. Document Conventions
      This document may use a scale to measure some requirements. For any requirement, a scale from 1 to 10 will be used to assess the importance of a feature such that 1 is the lowest level and 10 is the highest. Every requirement is dealt with as a separate unit that has its own evaluation criteria and priority.
    3. Intended Audience and Reading Suggestions
      The targeted audience to read this document are the stakeholders of the project which includes the project manager, ML experts, UI/UX designers, web/app developers, database administrators, back-end and front-end developers, marketing team, finance team, Research and Development team, and the end-users, VCs. In order to grasp the full scope of all requirements, read this document from top to bottom. Please note that some sections can be more important to some stakeholders while other sections are not.
    4. Product Scope
      The purpose of Fortune is to accelerate the funding process between VCs and startups. The vision of this project is to maximize the benefits from startups data and utilize it in an effective machine learning model. Our mission is to create a platform that aids VCs by applying data analytics on startups’ data and providing the possibility of their success. The corporate goal behind this project is to help VCs make better investment decisions and aid startups by getting them the resources they need. This platform will be implemented as a web application that is user friendly and reliable for VCs to use.

  2. Overall Description

    1. Product Perspective
      This product is a new self-contained product. It is not a part of a bigger system nor a replacement of an existing product. Our service is going to have a reliable network and database that will be efficient enough to cover all users.
    2. Product Functions
      User Registration/Sign Up
      Sign In/Log In
      Selection between Entrepreneur and Investor Modes
      Creating New Startup Record
      Evaluating Startup through the ML Model
      Explore new investment opportunities
      Create/Modify a Watchlist of Interesting Startups
      Costumer Service
    3. User Classes and Characteristics
      1. Administrators
        All available permissions and privileges are granted.
      2. Developers
        Full access to the application source code.
        Commit changes to oneself's branch.
        CRUD operations on thd database.
      3. Investors
        Ability to browse available startups.
        Ability to contact startup accounts to reveal their predicted SCORES???!!! that are evaluated through the ML model running in the backend.
      4. Entrepreneurs
        Will be provided with the ability to run the ML model on their startup's accounts.
        Browse through other available startup accounts.
      5. General Users
        Browse through available startup accounts.
    4. Operating Environment
      Fortune will be working on any devices that can access the internet. As it’s a web app, The website will be working on the common web browsers (chrome for windows and safari for mac users). Some features may not be supported by depreciated browsers like internet explorer.
      The software does not have any hardware requirements. All the needed processing power will be accessed remotely by the web application.
    5. Design and Implementation Constraints
      Fortune will not be able to verify the accuracy and authenticity of the data entered by the entrepreneur user class. However, a reporting functionality will be available for possible deletion of the users' data of any class, and that will be done by the administrators accounts.
    6. User Documentation
      The absolute first time a client joins the framework, there can be help prompts in application or in the program to assist them with exploring the interface and the various capacities. They can quit the instructional exercise whenever by clicking “×”.
      There will be a “Help” button for the user to read the user manual.
      Terms and conditions will appear for the user while signing up for him/her to read it and agree on it to be able to use the application.
